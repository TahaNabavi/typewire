# Security Policy

## Supported versions

TypeWire packages follow semantic versioning. Security fixes are released against
the **latest published version** of each affected `@tahanabavi/*` package. Please
upgrade to the latest release before reporting.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report them privately through GitHub's built-in flow:

> **[Report a vulnerability →](https://github.com/TahaNabavi/typewire/security/advisories/new)**
> (Repo → **Security** tab → **Advisories** → **Report a vulnerability**)

If you cannot use GitHub's private reporting, email **sayyedtahanabavi@gmail.com**
with the details.

Please include, where possible:

- the affected package(s) and version(s),
- a description of the vulnerability and its impact,
- steps to reproduce or a proof-of-concept,
- any suggested mitigation.

## What to expect

- **Acknowledgement** within a few business days.
- An assessment of the report and, if valid, a plan and timeline for a fix.
- A coordinated disclosure — we'll credit you in the advisory unless you prefer to
  remain anonymous.

Thank you for helping keep TypeWire and its users safe.
